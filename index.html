>Сканер уязвимостей сайтов

Пользователь должен ввести URL сайта, указать глубину просмотра страниц на этом сайте. Решение должно провести проверку страниц на уязвимости XSS, SQL-инъекции, IDOR.

>Декодер строк

Пользователь должен ввести строку, которая является закодированной версией послания. Решение должно проверить эту троку на все доступные методы кодирования, будь то BASE64, доступные таблицы MD5, ASCII и пр. и выдать ответ, содержащий расшифрованное послание и метод, которым данное послание было закодировано. К одной строке может применяться несколько методов шифрования.

>Сервис для валидации реального email

Необходимо реализовать сервис, который сможет распознавать временный email созданный в целях мошенничества. Такие email как правила используют для регистрации временных - фальшивых аккаунтов в соц. сетях.

>Сервис для определения буллинга в соц. сетях

Сервис должен детектировать наличие "травли" в соцсетях. Как вариант можно реализовать сервис в виде api, на входе сервис принимает сообщение, на выходе информацию о наличии буллинга.

>API для работы с банковскими картами от сторонних разработчиков

API должно предоставлять данные по авторизации (может быть как hash, так и авторизация, либо иной вариант).

Функции:

  • Авторизация пользователя
  • Запросить список банковских карт, доступных пользователю
  • Запросить детальную информацию по банковской карте, доступной пользователю: PAN xxxx xxxx xxxx 1234, месяц и год истечения, текущий баланс
  • Запросить историю движения денежных средства по банковской карте, доступной пользователю: дата/время, тип операции (поступление, списание), сумма операции, контрагент
